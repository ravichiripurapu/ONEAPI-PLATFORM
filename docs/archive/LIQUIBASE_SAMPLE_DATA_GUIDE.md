# Liquibase and Sample Data Setup Guide

## Overview

This guide explains the Liquibase configuration and sample data generation setup for the OneAPI Platform.

## What's Been Configured

```xml

</dependency>

<!-- JavaFaker -->
<dependency>
    <groupId>com.github.javafaker</groupId>
    <artifactId>javafaker</artifactId>
    <version>1.0.2</version>
</dependency>
```

#### Security App (`oneapi-security-app/pom.xml`)

### 3. Sample Database Tables

Five interconnected sample tables are created automatically:

#### `sample_employees`
Employee records with realistic data including:
- Employee ID, name, email, phone
- Department, position, salary
- Hire date, manager relationship
- Active status

#### `sample_customers`
Customer records with:
- Customer ID, company name, contact info
- Full address (street, city, state, country, postal code)
- Credit limit, customer since date
- Active status

#### `sample_products`
Product catalog with:
- Product ID, name, category
- Description, unit price
- Stock quantity, reorder level
- Supplier, availability status

#### `sample_orders`
Order records with:
- Order ID, customer reference
- Order, ship, and required dates
- Status (PENDING, PROCESSING, SHIPPED, DELIVERED, CANCELLED)
- Total amount, tax amount
- Shipping address, payment method

#### `sample_order_items`
Order line items with:
- Order and product references
- Quantity, unit price
- Discount, line total

### 4. Sample Data Generation

#### FakerDataService

Automatically generates realistic test data on application startup using JavaFaker library.

**Location:** `io.oneapi.admin.service.FakerDataService`

**Default Quantities:**
- 50 Employees
- 100 Customers
- 200 Products
- 300 Orders (with 1-5 items each)

**Features:**
- Realistic names, emails, addresses
- Valid relationships (employees have managers, orders have customers/products)
- Proper data types and constraints
- Automatic execution on startup (if tables are empty)
- Transaction support for data integrity

#### Data Generation Logic

```java
@EventListener(ApplicationReadyEvent.class)
@Transactional
public void initializeSampleData() {
    // Only generates if tables are empty
    // Creates employees → customers → products → orders → order items
    // Logs statistics after completion
}
```

### 5. REST API Endpoints

#### Sample Data Management

**Base URL:** `http://localhost:8090/api/sample-data`

##### Regenerate Sample Data
```bash
POST /api/sample-data/regenerate
```
Clears existing data and generates fresh sample data.

**Response:**
```json
{
  "status": "success",
  "message": "Sample data has been regenerated successfully"
}
```

##### Clear Sample Data
```bash
DELETE /api/sample-data/clear
```
Removes all records from sample tables.

**Response:**
```json
{
  "status": "success",
  "message": "Sample data has been cleared successfully"
}
```

##### Get Sample Data Info
```bash
GET /api/sample-data/info
```
Returns information about sample tables.

**Response:**
```json
{
  "tables": [
    "sample_employees",
    "sample_customers",
    "sample_products",
    "sample_orders",
    "sample_order_items"
  ],
  "description": "Sample database tables for testing OneAPI query functionality",
  "autoGenerated": true,
  "generatedOn": "Application startup (if tables are empty)"
}
```

### 4. Query Sample Data

#### Using H2 Console

```sql
-- View employees
SELECT * FROM sample_employees LIMIT 10;

-- View customers by country
SELECT country, COUNT(*) as count
FROM sample_customers
GROUP BY country
ORDER BY count DESC;

-- View top products by sales
SELECT p.product_name, p.category, SUM(oi.line_total) as total_sales
FROM sample_products p
JOIN sample_order_items oi ON p.id = oi.product_id
GROUP BY p.id, p.product_name, p.category
ORDER BY total_sales DESC
LIMIT 10;

-- View orders with customer details
SELECT o.order_id, o.order_date, o.status, o.total_amount,
       c.company_name, c.contact_name
FROM sample_orders o
JOIN sample_customers c ON o.customer_id = c.id
ORDER BY o.order_date DESC
LIMIT 20;
```

#### Using OneAPI Database Query Service

First, create a database connection pointing to the H2 database:

```bash
curl -X POST http://localhost:8090/api/connections \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Sample DB",
    "type": "H2",
    "host": "localhost",
    "port": 9092,
    "database": "./data/oneapi-admin",
    "username": "sa",
    "password": ""
  }'
```

Then query the sample tables using the DatabaseQueryService endpoints.

## Testing Scenarios

### Scenario 1: Basic Queries

Test simple SELECT queries on any sample table:
```sql
SELECT * FROM sample_employees WHERE department = 'Engineering';
SELECT * FROM sample_products WHERE category = 'Electronics';
SELECT * FROM sample_orders WHERE status = 'DELIVERED';
```

### Scenario 2: Complex Joins

Test multi-table joins:
```sql
SELECT
    o.order_id,
    c.company_name,
    p.product_name,
    oi.quantity,
    oi.line_total
FROM sample_orders o
JOIN sample_customers c ON o.customer_id = c.id
JOIN sample_order_items oi ON o.id = oi.order_id
JOIN sample_products p ON oi.product_id = p.id
WHERE o.status = 'DELIVERED'
ORDER BY o.order_date DESC;
```

### Scenario 3: Aggregations

Test aggregate functions:
```sql
-- Sales by category
SELECT
    p.category,
    COUNT(DISTINCT oi.order_id) as order_count,
    SUM(oi.quantity) as total_quantity,
    SUM(oi.line_total) as total_sales
FROM sample_products p
JOIN sample_order_items oi ON p.id = oi.product_id
GROUP BY p.category
ORDER BY total_sales DESC;

-- Employee count by department
SELECT department, COUNT(*) as employee_count
FROM sample_employees
WHERE is_active = true
GROUP BY department
ORDER BY employee_count DESC;
```

### Scenario 4: Pagination Testing

Test query pagination with large result sets:
```sql

SELECT * FROM sample_customers LIMIT 50 OFFSET 0;

-- Page 2 (next 50 records)
SELECT * FROM sample_customers LIMIT 50 OFFSET 50;
```

## Configuration

### Modify Data Quantities

Edit `FakerDataService.java` constants:

```java
private static final int NUM_EMPLOYEES = 50;   // Change to desired count
private static final int NUM_CUSTOMERS = 100;  // Change to desired count
private static final int NUM_PRODUCTS = 200;   // Change to desired count
private static final int NUM_ORDERS = 300;     // Change to desired count
```

### Disable Liquibase (if needed)

In `application.yml`:
```yaml
spring:
  liquibase:
    enabled: false
```

### Disable Auto-Generation

Comment out the `@EventListener` annotation in `FakerDataService.java`:
```java
// @EventListener(ApplicationReadyEvent.class)
@Transactional
public void initializeSampleData() {
    // ...
}
```

## Troubleshooting

### Issue: Tables Not Created

**Solution:** Check Liquibase logs on startup. Ensure `db.changelog-master.xml` is in the correct location.

### Issue: No Data Generated

**Solution:**
1. Check if tables already have data (service only runs if empty)
2. Look for errors in FakerDataService logs
3. Manually trigger regeneration via API

### Issue: Foreign Key Violations

**Solution:** Use the regenerate endpoint which properly clears data in dependency order.

### Issue: Build Failures

**Solution:** Ensure all dependencies are downloaded:
```bash
mvn clean install -U
```
